360Hub Demo

Introduction

This virtual reality demonstration seeks to reinforce the need for branded - or at least content specific - hubs for related 360 video content.  

Process

360 video is rapidly becoming cheaper and easier to make, with visual and audio quality gaining by leaps and bounds.  Soon, 6DoF will even be possible thanks to light field cameras being developed.  However, most of the time it's accessed on channels of mass distribution like YouTube and Facebook.  While this is the easiest way to distribute, it may not be the best.  As I seek to show in the demo, having a viewer inhabit an interactive space that you control completely could be more valuable than clicks from a distracted viewer.

Build

This project was built with Unity5.5.0f3 and Google VR SDK 1.0.  Because the idea was to open on a 360 video, I had to find a way to play an equirectangular video as a 360 sphere.  Luckily, that just what the team at Renderheads have done with the AVPro Video Unity Plugin.  I was able to painlessly project videos not only onto the inside of a sphere, but also onto a plane to simulate an LCD screen in a later scene.

Audio playback of voiceover

I wanted to use voiceover for each scene, and not just depend on UI elements like text and graphics.  However, I was having trouble with having each audio clip finish before starting the next.  How could I dynamically change the audio source present in the scene and play whatever voiceover clip I wanted and at given time? This is when I came across IEnumerator.  Usage here:

Remove
using System.Collections;
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.SceneManagement;

public class gameLogic_01 : MonoBehaviour {

    //created references to all of the voiceover clips
    public AudioClip vo0101;
    public AudioClip vo0102;
    public AudioClip vo0103;

    //created references to the UI elements in the scene
    public GameObject uiSamsung;
    public GameObject uiLytro;



    // Use this for initialization
    IEnumerator Start () {
        //accesses the audio source present on the game object and names it "audio"
        AudioSource audio = GetComponent<AudioSource> ();
        //sets "audio" to contain the vo0101 audio clip and plays it
        audio.clip = vo0101;
        audio.Play ();
        //The magic! This pauses the method as long as the clip is still playing
        yield return new WaitForSeconds(audio.clip.length);
        //Once the first vo is done, activate the second UI
        uiSamsung.SetActive (true);
        //replace vo0101 wtih vo0102 in AudioSource and play
        audio.clip = vo0102;
        audio.Play();
        //again, this will pause the method
        yield return new WaitForSeconds(audio.clip.length);
        //activates last UI element and vo clip
        uiLytro.SetActive (true);
        audio.clip = vo0103;
        audio.Play();
    
    
    
    }

    //Changes the scene
    public void GoToScene02 () {

        SceneManager.LoadScene ("Apartment", LoadSceneMode.Single);
    }
}

The way in which UI elements were used in the first 360 video scene remained basically unchanged in the Apartment scene.  The unique attribute here was moving the point of view around as UI elements were interacted with.  For example, the user clicks the dialogue box's button, and is then transported from a standing position to being seated at a desk.  For my purposes I didn't want a user to be able to move about freely because I wanted them to stay involved with my demonstration.  Any virtual reality developer will tell you motion is one of the more tricky things in this medium.  However, in a short time it has become best practice if you must move around that it be at a constant speed and be in quick bursts.  So, with that in mind I was able to emply a Unity plugin called iTween to quickly and easily call movements based on the users UI inputs.
